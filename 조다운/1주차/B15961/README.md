### 문제 이해 중

- 2차원 배열을 만들어서 [초밥 번호][개수] 로 관리하려고 했는데 초밥 가짓수가 3000 개라서 그러면 배열의 배열 → 메모리 레전드
- 매 구간마다 종류 다시 세면 시간 초과 → 슬라이딩 윈도우!

### 풀이 방법

- **앞 초밥 제거**: `rail[i]`
  - `visited--`
  - 0이 되면 그 초밥은 더 이상 윈도우에 없으므로 `cnt--`
- **뒤 초밥 추가**: `rail[(i+k) % n]`
  원형이므로 `% n`으로 이어준다.
  - 추가하기 전에 `visited==0`이면 새로운 종류이므로 `cnt++`
  - `visited++`

- **쿠폰 초밥(c) 적용**
  - 현재 윈도우에 없으면 추가로 먹을 수 있는 것
  - 추가 초밥은 연속될 필요는 없기 때문에 개수만 더함
