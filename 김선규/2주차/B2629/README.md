# ⚖️ BOJ 2629 - 양팔저울

- 🔗 문제 링크: https://www.acmicpc.net/problem/2629
- 🏷️ 분류: DP(가능 무게 집합), 집합(Set) 기반 상태 전이

---

## 📌 문제 요약
서로 다른 무게의 추 `N`개가 주어진다.  
각 추는 **왼쪽 저울 / 오른쪽 저울 / 사용 안 함** 중 하나로 놓을 수 있다.

구슬 `M`개에 대해, 각 구슬의 무게를 저울로 **정확히 측정할 수 있는지** `Y/N`으로 출력한다.

---

## 💡 풀이 아이디어 (가능한 무게 차이 집합)
양팔저울은 “양쪽 무게의 차이”로 측정 가능 여부가 결정된다.

어떤 시점에 만들 수 있는 무게 차이를 `set`에 저장한다고 하면,
새로운 추 무게 `w`를 추가할 때 가능한 경우는 3가지다.

1) ✅ 추를 한쪽에 올려 차이를 늘림  
- 기존 차이 `d` → `d + w`

2) ✅ 추를 반대쪽에 올려 차이를 줄임(절댓값)  
- 기존 차이 `d` → `|d - w|`

3) ✅ 추를 사용하지 않음  
- 기존 차이 `d` 유지

또한, 새 추만 단독으로 사용하는 경우도 가능하므로 `w` 자체도 추가한다.

즉, 매 추마다 `set`을 확장해 나가면 최종적으로 “만들 수 있는 모든 무게 차이”가 구해진다.  
구슬 무게 `x`가 `set`에 포함되면 측정 가능 → `Y`, 아니면 `N`.

---

## ✅ 상태 정의
- `set` : 현재까지의 추들로 만들 수 있는 “무게 차이”들의 집합  
  - 예: `set = {1, 3, 4, 7, ...}`

초기에는 아무 추도 없으므로 `set`은 비어있고,
추를 하나씩 보며 새로운 `newSet`을 만들어 업데이트한다.

---

## 🔁 전이(업데이트) 과정
새로운 추 무게 `inputNum = w`에 대해:

- `newSet`을 `set` 복사본으로 시작(사용 안 함 케이스 포함)
- 모든 `d ∈ set`에 대해:
  - `|d - w|` 추가
  - `d + w` 추가
- `w` 추가(추 단독 사용)
- `set = newSet`로 갱신

이 과정을 N번 반복한다.

---

## 🧾 쿼리 처리
각 구슬 무게 `query`에 대해:

- `set.contains(query)` 이면 `"Y"`
- 아니면 `"N"`

문제 출력 형식에 맞게 공백을 붙여 출력한다.

---

## ⏱️ 복잡도
- 각 추마다 `set`의 모든 원소를 한 번씩 순회하며 갱신
- `HashSet`의 평균 삽입/조회는 `O(1)`

따라서 대략:
- 시간 복잡도: `O(N * |set|)` (|set|은 가능한 차이 수)
- 공간 복잡도: `O(|set|)`

---

## 📝 구현 포인트(코드 기준)
- `newSet = new HashSet<>(set)`로 시작해 “추를 사용하지 않는 경우”를 자연스럽게 포함
- `Math.abs(num - inputNum)`으로 반대편에 올리는 경우를 처리
- 마지막에 `set.contains(query)`로 측정 가능 여부를 즉시 판단
