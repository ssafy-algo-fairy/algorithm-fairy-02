# 🚀 Algorithm Study

9명의 개발자가 함께하는 알고리즘 스터디 저장소입니다.  
매일 문제를 풀고 서로의 코드를 리뷰하며 함께 성장합니다.

---

## 📌 스터디 컨벤션 (Naming)
모든 명명 규칙은 **`이름-날짜-문제번호(B:백준,S:SWEA)`** 흐름으로 통일합니다.
| 구분 | 형식 | 예시 |
| :--- | :--- | :--- |
| **디렉토리** | `이름/주차/문제번호/` | `조의재/1주차/B12865/` |
| **브랜치** | `이름-날짜-문제번호` | `mokjak-260203-B12865` |
| **커밋** | `feat: 이름 날짜 문제번호` | `feat: 조의재 260203 B12865` |
| **PR 제목** | `[이름] 날짜 문제번호` | `[조의재] 260203 B12865` |

---

## 📂 디렉토리 구조
모든 작업은 개인 성함 폴더 내에서 진행합니다.

```text
 [이름]/
    └── [주차]/
        └── [문제]/
            ├── Main.java
            └── README.md (문제 설명 및 풀이 전략)

```
EX) `조의재/1주차/B12865/Main.java`

## 🛠 스터디 프로세스

### 1. 문제 풀기 전 pull & 브랜치 생성
항상 main 브랜치를 최신 상태로 만든 뒤, **새 브랜치**를 생성합니다. (날짜는 YYMMDD 6자리)
```bash
# main 브랜치로 이동 후 최신 코드 가져오기
git checkout main
git pull origin main

# 새 브랜치 만들기 및 이동 (백준:B, SWEA:S)
git checkout -b 이름-날짜-문제번호
```
EX) `mokjak-260203-B12865`

---

### 2. 코드 완성 후 push와 PR

```bash
# 변경 사항 추가 및 커밋
git add .
git commit -m "feat: 이름 날짜 문제번호"

# 원격 저장소에 내 브랜치 올리기
git push origin 브랜치명

```
**PR 생성 시 다음 설정을 수행해 주세요:**
* **Assignee**: 본인 선택 (담당자 표시)
* **Reviewers**: 본인 제외한 리뷰어 선택(선택한 사용자에게 리뷰 요청 메일이 갑니다) 
---
### 3. 코드 리뷰 및 머지(Merge)
동료의 코드를 확인하고 피드백을 주고받는 과정입니다. **최소 2명 이상의 Approve**가 있어야 머지가 가능합니다.
#### ✅ 리뷰 및 커멘트 다는 방법
1. **PR 목록 확인:** 상단 **[Pull requests]** 탭에서 리뷰할 대상(In Review 상태)을 선택합니다.
2. **코드 보기:** PR 페이지 상단의 **[Files changed]** 탭을 클릭합니다.
3. **커멘트 작성:** - 특정 코드 라인에 마우스를 올리면 나타나는 **[+] 버튼**을 클릭합니다.
   - 의견을 작성한 뒤 **[Start a review]** 를 누릅니다. (여러 줄에 남길 경우 계속 작성)
4. **리뷰 전송 (중요!):** - 모든 커멘트를 작성했다면 우측 상단의 **[Finish your review]** 혹은 **[Review changes]** 버튼을 누릅니다.
   - 아래 세 가지 중 하나를 선택하고 **[Submit review]** 를 클릭합니다.
     - 💬 **Comment:** 단순 질문이나 의견을 남길 때
     - ✅ **Approve:** 코드에 문제가 없고 머지를 승인할 때
     - ❌ **Request changes:** 반드시 수정이 필요한 부분이 있을 때

#### ✅ 머지(Merge) 진행
- **승인 확인:** 본인의 PR에 `Approve`가 2개 이상 쌓였는지 확인합니다.
- **머지 실행:** 모든 조건이 충족되면 PR 하단의 **[Merge pull request]** 버튼이 초록색으로 활성화됩니다. 클릭하여 머지를 완료합니다.
* **브랜치 삭제**: 머지가 완료된 브랜치는 로컬에서 삭제하여 목록을 관리합니다.(권장)
  * `git branch -d 브랜치명` (로컬 브랜치 삭제 명령어)

## ⚠️ 주의사항
* Force Push 금지: `git push -f`는 다른 사람의 코드를 지울 수 있으므로 절대 사용하지 않습니다.
* 폴더 주의: 본인의 폴더 외에 다른 사람의 폴더나 파일을 수정하지 않도록 주의해 주세요.
