## 📝 문제 요약

아래와 같은 방식으로 수식을 계산한다.

1. 수식에서 맨 앞의 연산자, 또는 맨 뒤의 연산자 먼저 계산한다. 단, 음수의 부호는 연산자로 취급하지 않는다.
2. 곱셈, 나눗셈을 덧셈, 뺄셈보다 더 먼저 계산한다.
3. 연산자의 우선순위가 같다면 해당 연산자를 계산했을 때 결과가 큰 것부터 계산한다.
4. 계산했을 때 결과 값 또한 같다면 앞에 것을 먼저 계산한다.

(단, 맨 앞을 제외하고는 음수가 들어오는 경우는 없다. 또 불필요한 0이 앞에 있을수도있다)

## 💡 접근 방법

- **알고리즘/자료구조: 덱**
- **핵심 아이디어: 덱**

# ✅ 풀이 과정

## 1️⃣ 문제 분석

- 숫자와 `+ - * /` 로 이루어진 수식이 주어진다.
- 일반적인 왼쪽부터 계산하는 방식이 아니다.
- 매 단계마다 **맨 앞 연산과 맨 뒤 연산 중 하나를 선택**해 계산한다.
- 선택 기준:
    1. **연산자 우선순위가 높은 것 먼저** (`/` > `+ -`)
    2. 우선순위가 같으면 **계산 결과가 더 큰 쪽 선택**
    3. 결과도 같으면 **앞쪽 선택**

즉,

> 매 단계마다 앞/뒤를 비교하며 동적으로 계산 방향을 결정하는 문제
> 

---

## 2️⃣ 해결 전략 수립

### 🔹 ① 수식 분리

- 숫자들은 하나의 덱(Deque)에 저장
- 연산자들도 별도의 덱에 저장
- 맨 앞이 음수일 수 있으므로 예외 처리

---

### 🔹 ② 반복 계산

연산자가 남아 있는 동안 반복:

1. 앞쪽 연산 후보 계산
2. 뒤쪽 연산 후보 계산
3. 우선순위 비교
4. 필요하면 결과값 비교
5. 선택된 쪽을 실제로 적용하고 덱을 갱신

이 과정을 반복하면

연산자는 하나씩 줄어들게 된다.

---

## 3️⃣ 핵심 아이디어

이 문제의 핵심은:

- **Deque를 사용해 앞/뒤 연산을 동시에 비교**
- 매 단계마다 "어디를 먼저 계산할지"를 판단하는 것
- 일반적인 스택 기반 계산 문제가 아니라
    
    → **양쪽을 동시에 고려하는 그리디 방식**
