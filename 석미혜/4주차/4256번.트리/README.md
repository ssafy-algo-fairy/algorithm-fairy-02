## 📝 문제 요약

전위순회 와 중위순회 결과를 보고 후위순회한 결과를 출력하라

## 💡 접근 방법

- **알고리즘/자료구조: 트리**
- **핵심 아이디어:**
- 후위 순회 (왼쪽 → 오른쪽 → 루트)
    1. 전위 순회 결과의 현재 위치(targetIndex)를 기준으로 루트 노드를 선택합니다.
    2. 타켓 노드를 기준으로 중위 순회한 결과를 `왼쪽`서브트리/ `오른쪽` 서브 트리로 나눠 다시 탐색합니다. ( 이때 전위 순회의 다음 루트를 사용하기 위해 targetIndex를 증가시킵니다.)
    3. 탐색 중 배열의 길이가 1이면 해당 노드를 출력합니다.
    4. 탐색이 완료되면 `루트`노드를 출력합니다.

## ✅ 풀이 과정

## 1. 문제 분석

- 전위 순회에서 첫 번째 값은 항상 **현재 서브트리의 루트**
- 중위 순회에서 해당 루트를 기준으로
    - 왼쪽 부분 → 왼쪽 서브트리
    - 오른쪽 부분 → 오른쪽 서브트리
- 우리가 구해야 하는 것은 **후위 순회 결과**

→ 전위 + 중위를 이용해 트리를 복원하지 않고

→ 재귀 분할만으로 후위 순회를 출력하는 문제

---

## 2. 해결 전략 수립

- `targetIndex`를 통해 현재 사용할 전위 순회 루트를 관리
- 중위 배열에서 해당 루트의 위치를 찾음
- 배열을 두 부분으로 나눔:
    - `0 ~ index-1` → 왼쪽 서브트리
    - `index+1 ~ 끝` → 오른쪽 서브트리
- 재귀 호출 순서:
    1. 왼쪽 탐색
    2. 오른쪽 탐색
    3. 루트 출력

이 순서가 바로 **후위 순회 순서**
