# BOJ 22860 - 폴더 정리

## 문제 설명

폴더와 파일의 계층 구조가 주어진다.  
각 항목은 부모 이름, 현재 이름, 타입으로 구성된다.

- 타입이 1이면 폴더
- 타입이 0이면 파일

이후 여러 개의 경로가 주어지며, 해당 경로 아래에 존재하는

- 서로 다른 파일의 개수
- 전체 파일의 개수

를 출력해야 한다.

---

## 해결 방법

### 1. 자료구조 설계

각 폴더와 파일을 Node 객체로 표현한다.

- name: 이름
- type: 1은 폴더, 0은 파일
- child: 폴더인 경우에만 자식 리스트를 가짐

파일은 자식을 가지지 않는다.

---

### 2. 트리 구성

1. main 폴더를 먼저 생성한다.
2. 입력을 n + m 번 받아 트리를 구성한다.
3. 부모가 아직 생성되지 않은 경우를 대비해 임시 저장 맵을 사용한다.
4. 입력이 끝난 후 한 번 더 순회하여 연결한다.

---

### 3. 경로 탐색

질의는 다음과 같은 형식이다.

main/FolderA/FolderB

문자열을 "/" 기준으로 분리한 뒤,

1. main에서 시작
2. 자식 리스트를 순차 탐색
3. 해당 이름을 찾으면 그 노드로 이동

이 과정을 반복하여 대상 폴더를 찾는다.

---

### 4. 파일 개수 계산

DFS 방식으로 재귀 탐색을 수행한다.

- 폴더이면 재귀 호출
- 파일이면:
  - 전체 개수 증가
  - HashSet에 추가하여 중복 제거

서로 다른 파일 개수는 HashSet의 크기이다.  
전체 파일 개수는 재귀 호출의 누적 값이다.

---

## 출력 형식

각 질의마다 다음 형식으로 출력한다.

서로다른파일개수 전체파일개수

---

## 시간 복잡도

- 트리 구성은 입력 개수만큼 수행
- 각 질의는 최악의 경우 전체 트리를 탐색

문제 제한 내에서 충분히 수행 가능하다.

---

## 개선 가능 사항

- 폴더 탐색을 List 대신 Map으로 관리하면 탐색 속도를 개선할 수 있다.
- 현재 부모 탐색은 선형 탐색으로 동작한다.

---

## 사용 개념

- 트리 구조
- DFS 재귀 탐색
- HashSet을 이용한 중복 제거
- equals 및 hashCode 오버라이딩
